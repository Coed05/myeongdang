# 우리 동네 안심 명당 — 백엔드 연산 엔진

산업단지 인근 아파트의 **소음·악취 도달 정도를 화공 수식으로 계산**해
종합 안심 주거 등급(AAA~F)을 산출하는 파이썬 백엔드입니다.

주소 입력 → 좌표 변환 → 주변 공장 5km 필터 → 기상(풍향·풍속) → 소음/악취 수식 → 등급화,
까지 전 과정을 모듈로 구현했습니다.

---

## 1. 설치

```bash
pip install -r requirements.txt
cp .env.example .env      # 그리고 .env 에 발급받은 인증키 입력
```

필요 인증키 3종 (모두 `.env` 에):
- `VWORLD_API_KEY` — 국토교통부 브이월드(vworld.kr) 지오코딩
- `DATA_GO_KR_API_KEY` — 공공데이터포털 산단공 입주기업 현황
- `KMA_API_KEY` — 기상청 초단기실황(풍향·풍속)

## 2. 바로 실행 (키 없이 데모)

```bash
python demo.py
```

가짜 공장 데이터(`sample_data.py`)로 엔진 전체를 돌려 콘솔 리포트를 출력합니다.
역할분담 문서의 "2일차 밤 검증 화면"에 해당합니다.

## 3. 실제 사용

```python
from factories import load_factories_from_api   # 또는 load_factories_from_csv
from pipeline import run_assessment, print_report

factory_df = load_factories_from_api("울산미포국가산업단지")
report = run_assessment(
    address="울산광역시 남구 야음동 OO아파트",
    factory_df=factory_df,
    use_live_weather=True,   # 기상청 실시간 풍향·풍속 사용
)
print_report(report)        # 콘솔 출력
# report 는 dict → 웹 프론트엔드의 '안심 주거 리포트 인포그래픽' 입력으로 사용
```

## 5. 구현한 수식 (제안서 그대로)

**소음 — 거리 역자승 감쇄**
```
SPL2 = SPL1 − 20·log10(r2 / r1)        (r1 = 공장경계 1m 기준)
```
**다중 공장 소음 — 로그 합산**
```
SPL_total = 10·log10( Σ 10^(SPLi/10) )
```
**악취 — 2차원 가우스 확산** (z=0 가정, 층수 배제)
```
C(x,y) = Q / (π·u·σy·σz) · exp(−y²/2σy²) · exp(−H²/2σz²)
```
- `x` = 풍하거리, `y` = 횡풍거리 (바람 방향으로 분해, `geo.decompose_wind`)
- `σy, σz` = 대기 안정도·풍하거리의 함수 (Briggs 식, `dispersion.py`)
- 풍상(x≤0)이면 도달 0 → 바람을 등진 아파트가 유리하게 평가됨

**종합 등급**
```
종합점수 = w1·소음점수 + w2·악취점수   (w1 + w2 = 1)
```

검증: `python -c "..."` 단위테스트로 역자승 6dB/배, 로그합산 +3dB, 풍상 0,
온축>횡풍, 하버사인 111km/도, 등급 경계를 모두 확인했습니다.

## 6. 파라미터 근거 (문서의 빨간 글씨 질문 대응)

- **5km 반경**: 생활소음·악취 민원이 통상적으로 집중되는 범위. `config.SEARCH_RADIUS_KM`
  에서 조정 가능 — 실서비스 전 측정/민원 데이터로 캘리브레이션 권장.
- **소음은 날씨 영향 거의 없음**: 음파는 거리·장애물이 지배 → 역자승만 적용.
- **악취는 풍향 지배**: 같은 거리라도 바람을 등지면(x≤0) 농도 0, 정면이면 최대.
  안정도(낮 불안정 A~C / 밤 안정 E~F)가 σ를 통해 농도에 반영됨.
- **층수 배제(z=0)**: 주거 '입지'를 추천하는 목적이라 1층이든 30층이든 동일 가정.

## 7. 알아두어야 할 한계 (정직한 캘리브레이션 과제)

1. **소음 r1=1m 기준**의 역자승은 점음원 가정이라 원거리에서 매우 빠르게 감쇄합니다
   (예: 85dB@1m → 약 1.5km에서 20dB대). 실제 공장은 분산음원이라 더 완만할 수 있어,
   기준거리/모델 보정이 필요합니다.
2. **업종별 Q·소음 dB 가중치는 초기 가정값**(상대 스케일)입니다. 절대 OU/dB를 법적
   기준과 직접 매칭하려면 환경공단 측정망·민원 통계로 보정해야 합니다 — 문서가 지적한
   "모델 유효성 검증" 단계.
3. **30일/연평균 등급**: 매일 바뀌는 날씨를 평탄화하려면 `run_assessment` 를 다양한
   풍향·풍속으로 반복 호출해 평균내는 래퍼를 추가하면 됩니다(다음 단계 권장).

## 8. 다음 단계 (화공팀 연동 후)

- 화공팀이 캘리브레이션한 **Q·SPL1·등급 기준표(엑셀)** 를 받아
  `config.INDUSTRY_WEIGHTS` 와 `grading.py` 임계값을 교체.
- 가격(국토부 실거래가) 결합 → 헤도닉 기반 가성비 지수 또는 가격·등급 슬라이더 필터.
- 프론트엔드: `run_assessment` 가 돌려주는 dict 를 그대로 지도/인포그래픽에 바인딩.
