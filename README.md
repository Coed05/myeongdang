# 우리 동네 안심 명당

**동네를 검색하면, 그 동네 아파트들의 소음·악취 안심 등급(AAA~F)을 매기고
가장 좋은 곳 TOP 3를 추천**하는 웹 서비스입니다. (현재 **울산·여수** 지원)

동네 입력 → 동네 좌표 변환 → 카카오로 동네 내 아파트 자동 수집 → 주변 산업단지 자동 탐지·공장 수집
→ 아파트별 5km 공장 필터 → 기상(풍향·풍속·하늘상태) → 소음/악취 화공 수식 → 변환점수·종합 등급
→ **TOP 3 추천 + 전체 아파트 등급표 + 규칙기반 AI 요약**, 까지 전 과정을 구현했습니다.

데모: <https://myeongdang.onrender.com>

## 1. 설치

```bash
pip install -r requirements.txt
cp .env.example .env      # 그리고 .env 에 발급받은 인증키 입력
```

필요 인증키 (모두 `.env` 에, **유출 주의 — 절대 커밋 금지**):
- `KAKAO_REST_KEY` — 카카오 로컬(주소·키워드 검색, **아파트 자동 수집**)
- `VWORLD_API_KEY` — 국토교통부 브이월드(vworld.kr) 지오코딩(보조)
- `DATA_GO_KR_API_KEY` — 공공데이터포털 산단공 입주기업 현황
- `KMA_API_KEY` — 기상청 초단기실황(풍향·풍속)

## 2. 실행

```bash
python api.py            # 로컬 개발 서버 (http://localhost:5000)
# 또는
gunicorn api:app        # 배포용
```

브라우저에서 열어 동네(예: `울산 남구 야음동`, `여수시 화양동`)를 검색하면 됩니다.
첫 조회는 공장 데이터 수집 때문에 느릴 수 있고, 같은 동네는 메모리 캐시로 빨라집니다.
공장 데이터는 `factories_prefetched.csv`(사전 수집본)가 있으면 즉시 사용합니다
(생성법은 `PREFETCH_GUIDE.md`, 배포는 `RENDER_DEPLOY.md`).

## 5. 구현한 수식

**소음 — 거리 역자승 감쇄 + 다중 음원 로그 합산**
```
SPL2 = SPL1 − 20·log10(r2)              (r1 = 공장경계 1m 기준, 거리 m)
SPL_total = 10·log10( Σ 10^(SPLi/10) )
```

**악취 — 2차원 가우스 확산** (z=0 가정, 층수 배제)
```
C(x,y) = Q / (π·u·σy·σz) · exp(−y²/2σy²) · exp(−H²/2σz²)
σy = a·x^b ,   σz = c·x^d + f           (Martin Table 3, x<1km / x≥1km 구간별 상수)
```
- `Q` [OU·m³/s] = 업종별 가중치(화학 100 / 플라스틱 60 / 금속 15) × 면적비
- `x`(풍하거리)는 km로 Martin 식에 대입, σ는 m. `y·H`는 m. `u`는 m/s 그대로.
- 풍상(x≤0)이면 도달 0 → 바람을 등진 아파트가 유리하게 평가됨.

**대기 안정도** — 고정값이 아니라 **풍속 + 하늘상태(기상청 SKY: 1 맑음/3 구름많음/4 흐림)** 로
Pasquill(1961) Table 2에 따라 A~F를 산정. 실시간 값이 없으면 울산 표준(낮 C / 밤 E)으로 폴백.

**통합 등급표(변환점수)** — 소음·악취를 비율로 합산하기 위해 각각 0~100점으로 환산.

| 등급 | 소음(dB) | 악취(OU) | 변환점수 |
|------|---------|---------|---------|
| AAA | ≤45 | ≤3 | 100 |
| AA | 46~51 | 4~6 | 85 |
| A | 52~57 | 7~10 | 70 |
| B | 58~63 | 11~15 | 55 |
| C | 64~69 | 16~20 | 40 |
| D | 70~75 | 21~25 | 25 |
| E | 76~85 | 26~30 | 10 |
| F | >85 | >30 | 0 |

```
종합점수 = w1·소음변환점수 + w2·악취변환점수   (w1 + w2 = 1)
→ 종합점수를 다시 AAA~F 등급으로 환산 (경계 = 인접 점수의 중간값)
```

## 6. 파라미터 근거

- **5km 반경**: 생활소음·악취 민원이 통상 집중되는 범위. `config.SEARCH_RADIUS_KM` 조정 가능.
- **소음은 날씨 영향 거의 없음**: 음파는 거리·장애물이 지배 → 역자승만 적용.
- **악취는 풍향 지배**: 같은 거리라도 바람을 등지면(x≤0) 농도 0, 정면이면 최대. 안정도가 σ를 통해 농도에 반영됨.
- **층수 배제(z=0)**: 주거 '입지'를 추천하는 목적이라 1층이든 30층이든 동일 가정.
- **울산·여수 한정**: 대규모 석유화학·산업단지가 밀집해 소음·악취 이슈가 뚜렷한 대표 지역으로 우선 적용.

## 7. 알아두어야 할 한계

1. **업종별 Q·소음 dB 가중치는 화공팀 산정값(상대 스케일)** 입니다. 절대 OU/dB를 법적 기준과
   직접 매칭하려면 환경공단 측정망·민원 통계로 보정해야 합니다.
2. **아파트 목록은 카카오 키워드 검색 기반**이라 일부 단지가 누락·중복될 수 있습니다.
3. **30일/연평균 등급**: 매일 바뀌는 날씨를 평탄화하려면 다양한 풍향·풍속으로 반복 호출해
   평균내는 래퍼를 추가하면 됩니다(다음 단계).
